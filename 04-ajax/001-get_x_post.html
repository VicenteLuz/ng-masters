GET x POST

GET e POST? O que é isso?

  • São os dois métodos mais usados pelo HTTP
  • (Sendo que o GET ganha disparado na frente)
  • Convenção: GET não deve alterar estado, pra "mexer no banco" vc deve usar POST
  • GET manda parâmetros na url?p1=v1&p2=v2
  • POST tipicamente manda parâmetros "por baixo dos panos"
  • POST também pode mandar parêmetros na URL, mas não faça isso!

Preciso obedecer essas convenções? Por que?

  • É bom obedecer sim!
  • Browsers podem cachear resultados de GETs
  • O backend tipicamente implementa proteção CSRF para POSTs

CSR o que???

  • CSRF == Cross-site request forgery. É uma proteção a mais contra roubo de sessão.
  • Cada nova página recebe um csrf_token diferente
  • Quando o navegador vai fazer um POST, tem que mandar o token junto.

O que isso tem a ver com Ajax?

  • Vc deve construir Apis AJAX que usem get e post de maneira adequada:
  • TwitterAPI.listTwittersOfUser(user) - isso precisa ser um GET
  • TwitterAPI.newTweet(text) - isso precisa ser um POST

Vc precisa de um jeito fácil de fazer gets e posts com AJAX.
